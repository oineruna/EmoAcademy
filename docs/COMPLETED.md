# EmoAcademy 完了したものの解説

最終更新: 2026-07-01

この文書は、これまで「完了したもの」と呼んでいた項目が、実際には何を意味するのかを説明するためのメモです。

## 1. ログイン/新規登録

### 完了した内容

- メールアドレスとパスワードで登録できる画面を作成。
- メールアドレスとパスワードでログインできる画面を作成。
- Googleログインボタンを追加。
- 学生/教師のロール選択を新規登録に追加。
- パスワード強度バー、パスワード表示切替、ログインしたままにするUIを追加。

### 意味

画面だけではなく、Supabase Authと接続する前提の実装まで入っています。Supabase側のURLと公開キーがVercelに設定されていれば、公開URLでもログインできます。

### まだ注意が必要なこと

Googleログインを使う場合は、Supabase側のRedirect URLsとGoogle Cloud Console側のOAuth設定が必要です。

## 2. 学生ダッシュボード

### 完了した内容

- Quizletに近い左メニュー、中央フィード、右側モニターのレイアウトを作成。
- 「続きから始める」「最近」「あなた向けの学習」「ライブラリー」「学習グループ」を配置。
- 日本語/英語の一時切り替えボタンを追加。
- 「質問・回答」を開始エリアに1つだけ戻し、質問送信と教師回答の確認画面へ移動できるようにしました。

### 意味

## 4. 感情モニター

### 完了した内容

- 右側に収まる感情モニターUIを作成。
- 閉じた状態とLIVE状態の見た目を同じカード構造に統一。
- モニターを開くとカメラが自動で開始し、リアルタイムに数値・棒グラフ・円上の点・タイムラインが動くようにしました。
- `C:\Users\Admin\Downloads\learning1-emotion_detection-main` の構成を参考に、カメラ、顔領域、Valence/Arousal、タイムライン更新の考え方を移植。
- Hugging Face上のEmotion APIに `enet_b0_8_va_mtl.pt` を配置し、モデルロード確認済み。

### 意味

ブラウザ側では即時に簡易推定を動かし、一定間隔でHugging FaceのEmotion APIにもフレームを送ってモデル推論します。動画そのものを保存するのではなく、保存対象は `valence`、`arousal`、主な感情ラベル、信頼度などの数値ログです。

### 注意

これは学習支援用の補助信号です。成績評価、処罰、能力判定には使わない設計にします。
